面试流程

  • 电话面试 => 共享桌面面试 => 现场面试

共享桌面面试

  • 思考清楚再开始编码
  • 良好的代码命名和缩进对齐习惯
  • 能够进行单元测试

面试的3个环节

  • 行为面试（了解应聘者的过往经验）
  • 技术面试（现场写代码）
  • 应聘者提问环节（问面试官几个问题）
    • 可以问自已学习遇到的问题

注意点

  • 如果应聘者在面试的时候没有听清楚或者听懂面试官的问题，千万不要不懂装懂，答非所问

  • 当不确定面试官的问题的时候，应聘者一定要大胆地向面试官多提问，直到了解面试官的意图为止

回答的问题是

  • 详细
  • 有深度
  • 分点

要做的准备

  • 自我介绍 30秒 到 1min 介绍 主要学习、工作经验
  • 准备要问答面试官的问题、
    • 关于公司的人员的架构
    • 自已做项目遇到的问题
    • 技术的探讨上的深度问题
  • 回答简历的项目点
  • 项目经验介绍 STAT 模式

为什么要跳槽？避免以下原因

  • 老板太苛刻

  • 同事太难相处

  • 加班太频繁

  • 工资太低

  • 公司的在业务上的技术栈太安逸了，技术没有得到提升

5种重要的素质

面试要的准备

  • 对编程语言、数据结构和算法等基础知识有全面的了解

  • 简单问题、注意细节、写出完整、鲁棒的代码

  • 复杂问题、通过画图、举具体例子分析和分解复杂问题等方法先理清思路再动手编程

  • 不断优化时间效率和空间效率，力求找到最优的解法

  • 应聘者应该主动提问，弄清楚题目的要求，表达自已的沟通能力 一个到两个问题

    • 不要问薪资
    • 不要问面试结果
    • 推荐问的问题是与应聘职位或者项目相关的问题

  • 扎实的基本知识：编程语言、数据结构和算法

  • 高质量的代码、鲁棒、稳定的大型软件

  • 清晰的思路

  • 优化效率的能力

  • 优秀的综合能力，编程能力和技术功底 沟通能力 和学习能力 知识迁移能力

面试难题的解决方法

    • 画图能使抽象问题形象化
    • 举例使抽象问题具体化
    • 分析使复杂问题简单化

全绿的 Github 提交记录

大公司基本上都有自己的招聘简历平台，刷简历时如果是 PDF 格式，
简历不过是你扎实技术基础和良好学习态度后的一道助力

1、技术栈描述请不要使用“熟悉、了解、精通…”等词汇，不用使用百分比形容对技术栈的熟悉程度。
2、如果照片颜值能为你加分的话可以放上自己的美照；否则不建议放，特别是着正装的证件照（毕竟生活中很少程序猿着正装）
3、不要自我评价中写有良好的沟通能力，认真负责，自学能力强等大家管用的词汇
4、技术栈中不要写Ajax、jQuery、BootStrap
5、培训机构出来的可以不要写这一条，没有别的意思，业界很多对培训机构有一定看法
6、简历尽量保持在一页有余、两页不满
7、简历务必使用pdf格式，常收到word格式等mac电脑无法打开，或者乱码

  • 技术名词注意书写正确，bable 是啥，一看就不熟悉啊

  • 技术名词注意大小写，JavaScript / HTML / CSS / Vue.js 等等

  • 写着熟悉 xxx，精通 xxx，真的熟悉、精通吗

「一页足矣」：10 ~ 30秒的时间，可能你的第二页简历永远不会被看到

「留白原则」：留白是中国艺术作品创作中常用的一种手法，在简历里也适用，每一行文字、每个模块之间都需要有一些间隔，使全篇更为协调和突出重点

「标准字体」：宋体即可，别用那些花里胡哨的字体

「简历模板」：弱视觉效果，黑白蓝灰四色即可，简单高效。但也别太简约啊，别用表格

写有大量与岗位不相关的各种获奖
“本人能吃苦，勤奋好学……”
2020年应届生：判断是否是真的实习生，很多已经毕业的人也会去面前端岗位，所以作为面试官，会去确认是否是“真的”实习生（法律规定，实习岗仅向未毕业的学生开放）
本科：判断是否符合有学历要求的企业
@gmail.com：判断是否会“科学上网”，是否是个“合格”的程序猿
求职意向：嗯，是web开发岗位
姓名：其实姓名无所谓，面试官在获取简历信息过中并不关心你叫什么
面试前端岗位，面试官不太会关注你是否熟练PS、BootStrap、jQuery（不是Jquery）、sql、Java、Ajax、编辑器等等。这些内容只会干扰面试官，以至于在技能情况这一栏眼睛就停留2s-4s钟就跳过。也就是说这里写的都是无用的信息，甚至让你减分（简历写的好坏也属于你综合能力好坏的体现！）。
使用了jQuery、BootStrap之类的项目建议不写。建议使用主流技术栈将老项目重构一遍，不然该项目经历出现在简历中是没有价值的。

没有项目经历

  • 可以放一些仿站，比如模仿一下zhihu、bilili、juejin或者国外某些高大上的网站。使用一些主流的技术栈去改造。（ps：github或一些技术论坛上有很多这类仿站教程可以去学习。）

项目数量和描述

  • 项目经历请保持在两个以上，讲究一主多辅，要保证第一个是最拿的出手的。这样就能让面试官在今后face to face过程中能聊聊你的这个项目，这就是你的谈资！解决了哪些难点，项目中有哪些技术手段实现的亮点，可以再附上一些项目结束后的人生或工作感悟。
